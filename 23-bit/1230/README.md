## 💡 접근 방식

### 1. 사용 알고리즘
* **자료 구조 (단일 연결 리스트)**: 문제에서 요구하는 삽입, 삭제, 추가 연산을 효율적으로 처리하기 위해 **단일 연결 리스트(Singly Linked List)**를 직접 구현하여 사용합니다.

### 2. 문제 풀이 과정
1.  암호문을 저장할 `Node` 클래스와, 이 노드들을 관리할 `SinglyLinkedList` 클래스를 정의합니다.
2.  `SinglyLinkedList` 클래스 내부에 문제의 요구사항에 맞게 특정 위치에 노드를 삽입하는 `insert(data, position)`, 특정 위치의 노드를 삭제하는 `delete(position)`, 리스트의 맨 뒤에 노드를 추가하는 `append(data)`, 특정 위치의 데이터를 조회하는 `get(position)` 메서드를 구현합니다.
3.  원본 암호문 뭉치를 입력받아 `append` 메서드를 반복 호출하여 초기 연결 리스트(`corpus`)를 생성합니다.
4.  전체 명령어 리스트를 입력받은 후, `while` 루프와 `index` 변수를 사용하여 명령어 리스트를 순차적으로 파싱합니다.
5.  명령어가 'I'(삽입)인 경우:
    * `x` (위치), `y` (개수)를 파싱합니다.
    * `y`번 반복하면서 `insert(si, x + i)`를 호출하여, `x`번째 위치부터 차례대로 `y`개의 암호문(`si`)을 삽입합니다.
6.  명령어가 'D'(삭제)인 경우:
    * `x` (위치), `y` (개수)를 파싱합니다.
    * `y`번 반복하면서 `delete(x)`를 호출합니다. (하나를 삭제하면 뒤의 노드가 해당 인덱스(`x`)가 되므로, 항상 `x`번째 노드를 `y`번 삭제합니다.)
7.  명령어가 'A'(추가)인 경우:
    * `y` (개수)를 파싱합니다.
    * `y`번 반복하면서 `append(si)`를 호출하여 리스트의 맨 뒤에 `y`개의 암호문을 덧붙입니다.
8.  모든 명령어 처리가 끝난 후, `get(x)` 메서드를 0부터 9까지 호출하여 최종 암호문 뭉치의 맨 앞 10개를 결과로 출력합니다.
